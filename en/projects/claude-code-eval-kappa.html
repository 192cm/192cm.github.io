Summary

This project compares human evaluation and LLM-as-Judge evaluation for Korean paper summaries generated by a Claude Code-built summarization workflow. Ten arXiv abstracts were evaluated under no-rubric, criteria, and rubric-based conditions, then measured with Cohen's kappa and quadratic weighted kappa.

Contents 1. Introduction 2. Target and Questions 3. Dataset 4. Method 5. Results 6. Analysis 7. Limits and Conclusion

1. Introduction

LLMs are widely used for open-ended generation tasks such as paper summarization, question answering, code generation, and agentic workflows. These tasks are difficult to evaluate with accuracy or exact match alone. A useful summary must be judged across factuality, faithfulness, usefulness, fluency, and format compliance.

This project operationalizes the vague idea of "summary quality" through explicit criteria and score anchors. The goal is to test whether human evaluation and LLM-as-Judge scoring move in the same direction, and whether rubric design improves agreement.

2. Evaluation Target and Research Questions

The target is a paper summarization system built with Claude Code. It takes a paper title and abstract, then generates a five-section Korean summary. Because the output is not a closed-form answer, string matching or ROUGE alone is not a sufficient evaluation method for the project goal.

2.1 Summary Format

  • Key contribution: the problem and the paper's main proposal
  • Methodology: models, experiments, and approach
  • Results: findings visible from the abstract
  • Limitations: constraints that can be inferred from the abstract
  • Keywords: terms for finding the paper again

2.2 Research Questions

  • How much do humans and LLM-as-Judge agree under a plain 1-to-5 score?
  • Does providing criteria improve agreement?
  • Does a score-anchored rubric make LLM judgment closer to human evaluation?
  • How do unweighted Cohen's kappa and quadratic weighted kappa differ?
  • Which quality dimensions explain disagreement cases?

3. Evaluation Dataset

The evaluation set contains ten arXiv papers sampled for the actual use case of the tracker. Each case has an ID from P01 to P10, with arXiv ID, title, matched keyword, and abstract stored in data/papers_sample.csv.

Topics include LLM transparency, MoE calibration, Bayesian in-context learning, on-device LLM serving, tool-calling agents, MLLM bias, SAR multimodal datasets, multilingual code benchmarks, coding-agent guidance, and safety-aligned LLMs.

Generated Korean summaries were stored in data/generated_outputs.csv. Evaluators judged only summary quality grounded in the abstract, excluding claims that would require the full paper.

4. Evaluation Method

4.1 Human Evaluation

Human evaluation was performed before seeing the LLM-as-Judge results to avoid anchoring the human scores. Scores and short rationales were recorded in data/human_ratings.csv.

4.2 LLM-as-Judge Evaluation

LLM-as-Judge scoring was performed with a ChatGPT-based GPT-5-series model. Each input contained only the paper title, abstract, and generated Korean summary. Human scores were excluded. The judge returned an integer score from 1 to 5 with a short rationale, recorded in data/llm_judge_ratings.csv.

4.3 Evaluation Conditions

  • No rubric: an intuitive 1-to-5 score for overall quality.
  • Criteria: one final score considering accuracy, completeness, usefulness, Korean fluency, and format compliance.
  • Rubric: explicit score anchors from 1 to 5 so evaluators judge against the same quality contract.

5. Evaluation Results

The no-rubric condition had the lowest agreement, with exact agreement of 0.40 and unweighted kappa of 0.0000. The human evaluator tended to give high scores when a summary was readable and captured the core idea, while the LLM judge penalized weak limitations or missing technical detail more strongly.

Criteria and rubric-based evaluation improved exact agreement to 0.60 and unweighted kappa to 0.3333. Once criteria were made explicit, both evaluators looked at more similar dimensions, especially factuality and format compliance. Boundary cases between 4 and 5, or 3 and 4, still remained.

5.1 Kappa Calculation

Human and LLM scores were merged into data/final_evaluation_dataset.csv. The script scripts/calculate_kappa.py calculates Cohen's kappa with sklearn.metrics.cohen_kappa_score and includes a manual fallback for environments without sklearn.

Because 1-to-5 scores are ordinal, both unweighted kappa and quadratic weighted kappa were reported. Weighted kappa treats adjacent score disagreements as smaller than distant disagreements.

6. Result Analysis

Every disagreement was only one point apart, so quadratic weighted kappa was higher than unweighted kappa. Criteria and rubric reached weighted kappa of 0.5556 compared with unweighted kappa of 0.3333. Even no-rubric scoring had weighted kappa of 0.3333 despite unweighted kappa of 0.0000.

This means the human and LLM judge were not judging in opposite directions. They mostly agreed on the direction of quality, but the LLM judge tended to be one step more conservative. For ordinal 1-to-5 evaluation, weighted kappa is therefore important for interpreting the real size of disagreement.

6.1 Disagreement Cases

  • P03 Bayesian In-Context Learning: the human saw the core idea and result as sufficiently conveyed, while the LLM wanted more detail about adaptation to new priors.
  • P06 StylisticBias: the human valued coverage of dataset size and main findings, while the LLM penalized the short limitation section.
  • P07 SARLO-80: the human valued dataset scale and composition, while the LLM wanted more explanation of SAR-specific use cases.
  • P10 Safety-Aligned LLMs: the human found the experiment and conclusion sufficient, while the LLM wanted more detail about refusal responses and context handling.

7. Limitations and Conclusion

The evaluation set is limited to ten cases, so more papers by topic would be needed for a more stable kappa estimate. The current setup also uses a single LLM judge; future work could add multiple judges or pairwise preference evaluation to reduce absolute-score anchor drift.

The most important observation is that the LLM judge is stricter than the human evaluator about technical completeness and limitation coverage. The human evaluator gave high scores when the summary was readable and useful, while the LLM judge penalized missing details from the abstract.

For open-ended generation systems, reporting criteria, rubrics, disagreement examples, and weighted kappa together is more persuasive than reporting only an average score.
